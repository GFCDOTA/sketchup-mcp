Maiores dificuldades do serviço MCP

  • Consensus é autoria HUMANA — não existe extrator PDF→consensus (raster+Hough fabricava parede falsa). Gargalo pra qualquer planta nova. produto
  • Escala precisa de âncora física — uma dimensão real conhecida vira o PT_TO_M; sem ela = BLOCKED (nunca chutar). extração
  • Fidelidade de abertura de janela — peitoril/verga; vidro errado no banho 2 corrigido recentemente. render
  • Multi-agent: colisão de worktree — sessões dividindo 1 worktree quase clobberam; orquestrador VÊ, isolamento PREVINE (follow-up). infra
  • Julgamento visual não é auto-confiável — negative_dogfood; depende de VISUAL_REVIEW humano. gate
  • Constantes do builder hardcoded — alturas/escala/crop p/ planta_74; blueprint pronto, espera árvore esfriar. generalização
Como o MCP usa o gate pra continuar: roteia decisões reais (9 triggers) pro :8765 em modo B; se uma sessão fica PARALYZED (cycle congelado) ou há consult sem resposta, o alarme dispara — aí use o Q&A.

Oportunidades & melhorias

  • Extrator vetorial PDF→consensus (walls = filled paths) — destrava generalização; só DEPOIS de ≥2 plantas no loop manual. produto
  • 2ª planta real como forcing function — prova a generalização com planta nova, não com abstração. produto
  • Swap-in das constantes do builder — executar o blueprint quando a árvore esfriar. generalização
  • Worktree-locking — fix-raiz da colisão multi-agent (chip aberto); o orquestrador só detecta. infra
  • Tier do oráculo — Opus+xhigh pras pesadas, Sonnet pra rotina (via multi-oracle). gate
  • Watchdog do gate — auto-restart se /health cair (hoje: hook de login + LIGAR-BRIDGE). infra
  • Consolidar serving pro repo canônico — sair do worktree wt-dash; aí o Lixão vê o runs/ scratch real. infra
  • RAG indexado — o file-fetch §6.3 é "RAG-bebê"; indexar depois do pipeline generalizado. enhancement
